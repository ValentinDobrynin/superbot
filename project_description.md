## What We Build

**SuperBot (vAIlentin 2.0)** — Telegram-бот, который имитирует стиль общения
владельца (Валентина) в групповых чатах. Бот читает историю чата, учится
стилю владельца, оценивает важность входящих сообщений и отвечает за него
там, где это уместно — так, чтобы участники чата не отличали бота от живого
человека.

Под капотом — `aiogram 3` (long-polling) + OpenAI (GPT-3.5 / embeddings)
+ PostgreSQL для хранения истории, тредов, тегов и стилевых профилей.

---

## For Whom

- **Основной пользователь — владелец бота** (`OWNER_ID`).
  Управляет ботом из личного чата с ним, видит статистику, настраивает
  поведение, загружает обучающие дампы.
- **Участники групповых чатов** — пассивные потребители: видят ответы бота
  как сообщения от Валентина, не управляют им.
- Бот **не является SaaS** для других пользователей: одна инсталляция —
  один владелец.

## Stack

- **Язык:** Python 3.11 (см. `render.yaml`; `pyproject.toml` допускает 3.8+,
  но прод и `make check` ориентируются на 3.11).
- **Telegram:** [`aiogram`](https://docs.aiogram.dev) ≥ 3.0 (long-polling,
  `MemoryStorage` для FSM).
- **БД:** PostgreSQL ≥ 13, доступ через `SQLAlchemy 2.x` (async) +
  `asyncpg`. Локально для тестов — `aiosqlite`.
- **Миграции:** Alembic (`src/database/migrations/`). В корне есть устаревшие
  ad-hoc SQL-скрипты в `migrations/` — их нужно мигрировать в Alembic
  (см. `TECH-002` в `backlog.md`).
- **LLM:** OpenAI SDK (async), модели `gpt-3.5-turbo` и
  `text-embedding-ada-002`.
- **Конфиг:** `pydantic-settings` + `.env`.
- **Деплой:** Render (background worker, Frankfurt), PostgreSQL в Render.
- **Тесты:** `pytest` + `pytest-asyncio` + `pytest-mock`.
- **Линт/формат/типы:** `black`, `isort`, `flake8`, `mypy`
  (настройки в `pyproject.toml` / `requirements-dev.txt`).

## Architecture Principles

1. **Тонкие хэндлеры, толстые сервисы.**
   `src/handlers/*` — только парсинг команды, проверка прав и вызов сервиса.
   Вся бизнес-логика и работа с OpenAI/БД — в `src/services/*`.

2. **Один источник сессии.**
   `DatabaseMiddleware` (`src/middleware.py`) пробрасывает `AsyncSession`
   во все хэндлеры через `data['session']`. Сервисы получают сессию аргументом
   и не создают её сами (исключение — фоновые задачи в `StatsService`,
   которые открывают свою сессию через `get_session()`).

3. **Owner-only management.**
   Все управляющие команды (`/status`, `/setmode`, `/refresh`, `/upload`, …)
   проверяют `message.from_user.id == settings.OWNER_ID` и работают только
   в личном чате с ботом. В группах бот только читает сообщения и (опционально)
   отвечает.

4. **Silent-by-default.**
   При добавлении бота в новый чат `is_silent=True`. Бот молча копит контекст,
   пока владелец явно не разрешит ответы (`/setmode`, `/set_probability`,
   `/smart_mode`).

5. **Два режима ответа на чат:**
   - **Probability mode** (`smart_mode=False`): отвечать с вероятностью
     `response_probability`.
   - **Smart mode** (`smart_mode=True`): GPT оценивает важность сообщения
     (0.0–1.0); отвечаем, если `importance ≥ importance_threshold`.
     Порог авто-подстраивается под целевой response rate 20–40%
     (`Chat.adjust_importance_threshold`).

6. **Стиль — артефакт обучения, не статика.**
   `Style` хранится в БД отдельно для каждого `ChatType`
   (`work` / `friendly` / `mixed`) и пересобирается командой `/refresh`
   из истории сообщений или загруженного дампа (`/upload`).

7. **Промпты в коде — временно.**
   Сейчас промпты живут в `openai_service.py` и `context_service.py`
   как f-строки. Целевое состояние — все промпты в `prompts/*.txt`
   (см. `TECH-001`).

8. **Глобальный shutdown — в памяти процесса.**
   `settings.is_shutdown` — runtime-флаг, не персистится. Перезапуск процесса
   сбрасывает его в `False`. Это сознательное решение: shutdown — аварийная
   ручка, не долгосрочный режим.

## What We Do NOT Do

- ❌ **Не делаем мульти-тенантность.** Один деплой обслуживает одного
  владельца. Никаких ролей, биллинга, разделения данных между владельцами.
- ❌ **Не пишем web UI / dashboard.** Все управление — через Telegram-команды
  владельца. `fastapi`/`uvicorn` в `requirements.txt` — мёртвый груз
  (см. `TECH-003`).
- ❌ **Не модерируем чат.** Бот не банит, не удаляет, не предупреждает.
  Он только читает и отвечает.
- ❌ **Не отвечаем самому владельцу.** Сообщения от `OWNER_ID` намеренно
  пропускаются в `process_message_and_respond`.
- ❌ **Не реализуем real-time стриминг ответов.** Ответы — одним сообщением
  через `message.reply`.
- ❌ **Не поддерживаем другие мессенджеры** (Discord, Slack и т.п.) —
  это в `Icebox`, не в активной разработке.
- ❌ **Не храним персональные данные дольше необходимого.**
  Тексты сообщений нужны только для контекста, статистики и обучения стилю.
  Любая интеграция с внешними аналитиками = отдельная задача с явным согласием
  владельца.
- ❌ **Не используем webhook-режим Telegram** — только long-polling
  (требование Render free tier и упрощение деплоя).
