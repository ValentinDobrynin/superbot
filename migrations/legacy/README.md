# migrations/legacy/

Здесь лежат **исторические** SQL-скрипты, которые когда-то применялись на
прод-БД руками, до того как проект полностью перешёл на Alembic.

⚠️ **Эти файлы НЕ применяются автоматически и не должны применяться на
живой БД.** Они оставлены только как исторический след / референс.

Канонический путь для всех новых изменений схемы:

- `src/database/migrations/versions/` — Alembic-ревизии.
- Конфиг — `src/alembic.ini`.
- Применение — `make migrate` (`alembic -c src/alembic.ini upgrade head`).

См. также задачу `TECH-002` в `backlog.md` — там описано, что эти
скрипты идемпотентны по содержимому и в текущем проде уже применены через
последовательность Alembic-ревизий апреля 2025 года.
